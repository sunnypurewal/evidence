# 60-second terminal capture script

Use this as the narration/script for a short terminal recording. It is written so the capture can be produced with any screen recorder or terminal GIF tool.

Target length: 55-60 seconds.

```sh
# 0s - show the project and help surface
git clone https://github.com/RiddimSoftware/evidence.git
cd evidence
swift run evidence -- --help

# 12s - show the app-owned config model
sed -n '1,120p' README.md

# 24s - show the screenshot-plan API
sed -n '1,120p' Sources/Evidence/ScreenshotPlan.swift

# 36s - show before/after PR evidence support
swift run evidence -- capture-pr --help

# 48s - show CI adoption path
sed -n '1,120p' Examples/workflows/capture-pr-on-pr.yml
```

## Caption

`evidence` turns real app runs into repeatable proof artifacts: screenshots, videos, manifests, and reviewer-facing reports from one open-source Swift package and CLI.

## Recording Checklist

- Use a clean terminal theme with 14-16pt monospace text.
- Keep the window around 120 columns wide so README and help output wrap cleanly.
- Pause briefly after each command so viewers can read the headline.
- End on the GitHub Action example so the call to action is concrete.
